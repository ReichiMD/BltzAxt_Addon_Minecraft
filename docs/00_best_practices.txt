--- GOLDEN RULES & CRITICAL SYNTAX (MINECRAFT BEDROCK 1.21+) ---
STRICTLY FOLLOW THESE RULES TO AVOID BUILD ERRORS.

1. FILE STRUCTURE & LOCATIONS (CRITICAL)
   * Behavior Pack (BP): Contains LOGIC (Items, Entities, Recipes).
     ✅ CORRECT: BP/items/my_item.json
     ❌ WRONG: RP/items/my_item.json (NEVER put item definitions in RP!)
   * Resource Pack (RP): Contains VISUALS (Textures, Models).
     ✅ CORRECT: RP/textures/items/my_icon.png

2. MANIFEST DEPENDENCIES (Auto-Install Fix)
   * The Behavior Pack (BP) MUST automatically load the Resource Pack (RP).
   * ACTION:
     1. Get the UUID from the Resource Pack's manifest header.
     2. Add it to the Behavior Pack's manifest under "dependencies".
     3. Ensure version numbers match.

3. ITEM DEFINITION SYNTAX (Format 1.21.0+)
   * Display Name: Must be an object, not a string.
     ❌ WRONG: "minecraft:display_name": "Super Axe"
     ✅ CORRECT: "minecraft:display_name": { "value": "Super Axe" }
   * Creative Menu: Use full identifiers.
     ❌ WRONG: "group": "sword"
     ✅ CORRECT: "group": "itemGroup.name.sword"
     (Valid groups: .axe, .pickaxe, .shovel, .hoe, .helmet, .chestplate, .leggings, .boots)

4. TEXTURE REGISTRATION
   * Placing a .png file is NOT enough. It must be linked in item_texture.json.
   * File: RP/textures/item_texture.json
   * Format:
     {
       "resource_pack_name": "vanilla",
       "texture_data": {
         "my_item_id": { "textures": "textures/items/my_texture_file_name" }
       }
     }
   * In Item JSON: "minecraft:icon": "my_item_id"

5. COMMANDS & FUNCTIONS
   * Always use the NAMESPACE in commands.
     ❌ WRONG: give @s emerald_axe
     ✅ CORRECT: give @s test:emerald_axe
   * For Starting Kits: Use simple .mcfunction files, not complex main.js scripts.
     File: BP/functions/give_kit.mcfunction

6. VANILLA ITEMS
   * Do NOT invent items.
   * ❌ WRONG: minecraft:ruby (Does not exist)
   * ✅ CORRECT: minecraft:emerald, minecraft:diamond

7. CREATIVE MENU CATEGORIES (Fix for "namespace prefix error")
   * In 1.21+, "menu_category" -> "group" MUST use full identifiers.
   * ❌ WRONG: "group": "axe"
   * ✅ CORRECT: "group": "itemGroup.name.axe"
   * ✅ CORRECT: "group": "itemGroup.name.sword"
   * ✅ CORRECT: "group": "itemGroup.name.pickaxe"

8. VERSIONING & STABILITY (CRITICAL RULE)
   * RECIPES: ALWAYS use "format_version": "1.12.0".
     Reason: Newer versions (1.20+) require complex "unlock data" logic. Version 1.12.0 works perfectly in 1.21+ and is instantly unlocked.
   * GENERAL PRINCIPLE: Prefer stable, older format versions for simple JSON files (Recipes, Loot Tables) unless modern features are explicitly requested.

9. CREATIVE MENU GROUPS (Fix for 1.21+ Errors)
   * CRITICAL: When defining "menu_category" -> "group", you MUST use the full internal name.
   * ❌ WRONG: "group": "sword"
   * ❌ WRONG: "group": "equipment"
   * ✅ CORRECT: "group": "itemGroup.name.sword"
   * ✅ CORRECT: "group": "itemGroup.name.axe"
   * ✅ CORRECT: "group": "itemGroup.name.equipment"
